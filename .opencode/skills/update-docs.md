# CrownFlow 文档更新 Skill

## 触发条件

当用户请求更新项目文档、同步文档状态、或使用 `/update-docs` 命令时触发。

## 任务说明

根据当前项目代码实际状态，更新以下两个文档：

1. `docs/DEVELOPMENT_PLAN.md` - 项目开发计划文档
2. `docs/FRONTEND_OPTIMIZATION_SUMMARY.md` - 前端实现说明文档

## 执行步骤

### 1. 分析项目现状

首先阅读以下文件了解项目结构：

- `frontend/src/pages/` 下所有页面组件
- `backend/app/api/` 下所有 API 路由
- `backend/app/models/models.py` 数据模型
- `frontend/src/hooks/` 自定义 Hooks
- `frontend/src/services/api.ts` API 封装

### 2. 更新 DEVELOPMENT_PLAN.md

文档应包含以下章节：

1. **项目简介** - 系统用途和核心功能
2. **技术架构** - 前后端技术栈、端口配置
3. **项目结构** - 目录树
4. **页面说明** - 每个页面的功能
5. **API 接口** - 所有接口列表
6. **数据模型** - Member 和 Category 字段
7. **支出计算逻辑** - 计算公式和趋势图规则
8. **快速开始** - 环境要求和启动步骤
9. **测试** - 测试命令

### 3. 更新 FRONTEND_OPTIMIZATION_SUMMARY.md

文档应包含以下章节：

1. **技术栈** - 表格形式
2. **目录结构** - 简洁的目录树
3. **页面实现** - 每个页面的功能和特点
4. **自定义 Hooks** - 列表说明
5. **API 服务层** - 请求特性和 API 模块
6. **性能优化** - 优化措施
7. **错误处理** - 处理方式
8. **环境配置** - 环境变量
9. **服务脚本** - 脚本命令

## 写作规范

1. **只写现状** - 描述当前已实现的功能，不要写计划中的功能
2. **简洁明了** - 使用表格和列表，避免冗长描述
3. **准确无误** - 确保技术细节与代码一致
4. **格式统一** - Markdown 格式规范

## 输出要求

- 直接覆写原有文档
- 文档末尾注明最后更新日期（格式：YYYY-MM-DD）
- 不要添加"后续扩展"、"TODO"等计划性内容
